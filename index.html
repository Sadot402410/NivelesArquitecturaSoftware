What is Software Architecture?

The software architecture of a computer program or system is the structure or structures of the system, which includes the software components, their visible external properties, and the relationships between them.


Level 1
  • Monolithic Architecture

    A monolithic architecture is a traditional model of a software program that is compiled as a unified unit and that is self-contained and independent of other applications. The word "monolith" often conjures up something big and icy, an image that's not far from the reality of monolithic architectures for software design. A monolithic architecture is a single, large computer network, with a code base that brings together all business interests.

  • Advantages

    Easy deployment, development, performance, simplified testing, easy debugging

  • Disadvantages

    Slower development speed, scalability, reliability, barrier to technology adaptation, lack of flexibility, implementation.

  • Example

    Every time you install a database on your system and access it to practice queries

  • Vulnerabilities

    Line of business (LOB) applications such as accounting and customer management system.

    Corporate web applications and websites.



Level 2
  • Client-Server

    The client-server architecture is a software design model. Here, the tasks are divided between the providers of resources or services, called servers, and the applicants, called clients.

  • Advantages

    centralization, security, easy to install, separation of responsibilities, portability

  • Disadvantages

    Updates (clients), concurrency, all or nothing, low level protocols, debugging.

  • Example

    An example of a client is Chrome, which is executed on our computer. Instead, the server is the place that you ask for data to display to the user.


Level 3
  • Model View-Controller (MVC)

    Model View Controller (MVC) is a style of software architecture that separates an application's data, user interface, and control logic into three distinct components.

  • Presentation Level

    The presentation layer is the user and communication interface of the application, where the end user interacts with the application. Its main purpose is to display information to the user and collect data from it.

  • Application Level

    The application tier, also known as the logical or middle tier, is the core of the application. At this tier, information collected at the presentation tier is processed, sometimes with other information at the data tier, through business logic; a specific set of business rules.

  • Data Level

    The data tier, sometimes called the database tier, data access tier, or back-end, is where the information processed by the application is stored and managed.

  • Advantages

    easy organization, adaptable to different frameworks, can be easily scaled, facilitates teamwork

  • Disadvantages

    It can be a complicated pattern since you don't usually use simple programs and it requires a large number of files or folders

  • Example

    Line of business (LOB) applications such as accounting and customer management system.

    Corporate web applications and websites.

  • Vulnerabilities

    Use a POS system to control inventories, users, sales, customers, suppliers, among others, and thus reduce in the short and medium term the different IT risks, as well as situations involving information, which may be accidental or provoked, such as alterations, unauthorized access, loss of vital information for consumers, such as names, account numbers, expiration dates of the same, for which they may be victims of theft, also affects the continuity and control of the business or seller's good name and other vulnerabilities
